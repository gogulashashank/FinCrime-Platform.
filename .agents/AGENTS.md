# Workspace Rules for FinCrime-Capstone

- Always optimize for portfolio credibility in AML, fraud, compliance analytics, and RegTech.
- Prefer business realism over academic novelty.
- Keep code modular, typed where practical, and easy to explain in interviews.
- Every module must contribute to a believable financial crime workflow.
- Do not introduce unnecessary frameworks.
- Before major code changes, state the plan.
- After major code changes, summarize files created and next steps.
- Keep README and docs updated continuously.
- Use synthetic data only.
- Design for UK banking/fintech relevance first, then broader global applicability.
- Where there is a choice, favor features that show SQL/Python/analytics/investigation skill overlap.
- Create recruiter-friendly artefacts: screenshots, diagrams, sample outputs, CV bullets, and interview notes.
